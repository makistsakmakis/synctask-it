Ενότητα 1

Εισαγωγή & Ρόλοι

Το SyncFlow οργανώνει τα IT projects και τα καθημερινά tasks σε ένα σημείο, πάνω σε SharePoint.

Τι είναι το SyncFlow

Μια web εφαρμογή όπου καταχωρούνται, εγκρίνονται και παρακολουθούνται τα Projects (έργα με έναρξη, deadline και ομάδα RACI) και τα Tasks (επιμέρους εργασίες, αιτήματα, bug fixes). Τα δεδομένα ζουν σε SharePoint lists — ό,τι βλέπετε στην εφαρμογή είναι ορατό και στο SharePoint, και αντίστροφα (π.χ. σχόλια, συνημμένα).

Μία πηγή αλήθειας: καμία εργασία δεν χάνεται σε emails — όλα έχουν status, υπεύθυνο και ιστορικό.

Οι 4 ρόλοι

ΡόλοςΠοιος είναιΣε μία γραμμή
OWNERΑπλός χρήστης / αιτών (requestor)Ανοίγει projects, παρακολουθεί τα δικά του, βλέπει tasks των έργων του.
SUPERVISORManagerΕγκρίνει (υπογράφει) projects που εποπτεύει και tasks που του ανατίθενται για SignOff.
IMPLEMENTORResource / υλοποιητήςΕκτελεί tasks: ενημερώνει status, ώρες, ημερομηνίες, σημειώσεις.
ADMIN / COOΔιαχειριστήςΠλήρης έλεγχος: όλα τα δεδομένα, αναθέσεις, RACI, Overviews, διαγραφές.

Ο ρόλος αναγνωρίζεται αυτόματα από το Microsoft 365 login και τις λίστες Resources/Managers. Ο Admin διαθέτει και Role preview στο πλάι για να βλέπει την εφαρμογή «με τα μάτια» κάθε ρόλου.

Το μενού πλοήγησης

  • Welcome — προσωπικό dashboard (όλοι οι ρόλοι).
  • Tasks Overview / Projects Overview — εποπτικές οθόνες ADMIN μόνο.
  • Tasks / Projects — οι λίστες εργασίας (όλοι, με scoping ανά ρόλο).
  • Waiting for Signoff — ουρά υπογραφών ADMIN + SUPERVISOR.
  • ❓ Manual — αυτό το εγχειρίδιο.
Ενότητα 2

Βασικές Έννοιες: RACI, Kanban, Gantt

Τρία καθιερωμένα εργαλεία διοίκησης έργων που χρησιμοποιεί η εφαρμογή. Δύο λεπτά ανάγνωσης αρκούν.

RACI — ποιος κάνει τι σε ένα έργο

Το RACI ξεκαθαρίζει τους ρόλους της ομάδας ενός project με 4 γράμματα:

ΓράμμαΣημαίνειΜε απλά λόγια
ResponsibleΥπεύθυνος υλοποίησηςΑυτός που «βάζει τα χέρια» και κάνει τη δουλειά.
AccountableΤελικά υπόλογοςΈχει τον τελικό λόγο και την ευθύνη του αποτελέσματος — ένας ανά έργο.
ConsultedΣυμβουλευόμενοςΡωτιέται πριν από αποφάσεις (ειδικοί, stakeholders).
InformedΕνημερωνόμενοςΚρατιέται ενήμερος για την πρόοδο, χωρίς εμπλοκή στην εκτέλεση.
Καμία παρεξήγηση «ποιος το είχε αναλάβει» — οι ρόλοι είναι γραμμένοι πάνω στο έργο.

Στο SyncFlow τα RACI πεδία συμπληρώνονται από τη λίστα Resources και τα επεξεργάζεται μόνο ο Admin. Η πλήρης εικόνα φαίνεται στο RACI Matrix.

Kanban — η εργασία σε στήλες

Πίνακας με στήλες ανά κατάσταση (Not Started, In Progress, Completed…). Κάθε κάρτα είναι ένα project ή task. Η ροή της δουλειάς φαίνεται με μια ματιά: πού συσσωρεύονται εκκρεμότητες, τι έχει κολλήσει.

Not Started
→
In Progress
→
Completed

Στα Kanban του SyncFlow σέρνετε την κάρτα από στήλη σε στήλη (drag-n-drop) και το status ενημερώνεται αυτόματα — με αυτόματη εφαρμογή των κανόνων status.

Gantt — η εργασία στον χρόνο

Οριζόντιο χρονοδιάγραμμα: κάθε task είναι μια μπάρα από την έναρξη ως το due date. Δείχνει επικαλύψεις, φόρτο ανά περίοδο και καθυστερήσεις σε σχέση με το σήμερα.

Dashboard

Συγκεντρωτικοί δείκτες (KPIs): σύνολα, ενεργά, ολοκληρωμένα, εκπρόθεσμα, κατανομές ανά status/owner. Ιδανικό για γρήγορο status reporting.

Ενότητα 4

Projects

Η λίστα και η φόρμα των έργων — τι βλέπει και τι επεξεργάζεται ο κάθε ρόλος.

Λίστα Projects (grid)

Στήλες: #, Τίτλος (με εικονίδιο), Owner, Supervisor, Status, ON_GOING, ημερομηνίες, RACI (συντομογραφίες — τα 2 πρώτα + «+Ν», πλήρης λίστα στο hover). Διαθέτει chips ανά status, αναζήτηση, φίλτρα/ταξινόμηση ανά στήλη και Export Excel (βλ. Κοινές Λειτουργίες). Γραμμές με deadline στο παρελθόν σημειώνονται κόκκινες.

Ποια projects βλέπει ο καθένας στη λίστα
ADMINΌλα.
OWNERΜόνο τα δικά του (όπου είναι Owner).
SUPERVISORΜόνο όσα εποπτεύει.
IMPLEMENTORΜόνο τα projects των tasks του.
ΌλοιΕπιπλέον, τα ON_GOING projects είναι ορατά σε όλους ανεξαιρέτως (βλ. ενότητα 7).

Δημιουργία & φόρμα project

Νέο project ανοίγουν ο OWNER και ο ADMIN. Υποχρεωτικά: Title, Supervisor (επιλέγεται από όλα τα πρόσωπα του SharePoint), Deadline. Με την αποθήκευση ανοίγει αυτόματα email προς τον Supervisor με θέμα «<Τίτλος> Needs your signoff» — το στέλνετε από το δικό σας mailbox.

ΠεδίοΠοιος το επεξεργάζεται
Title, Product, Link, Notes, Proposed start, Deadline, Icon, SupervisorOWNER + ADMIN
Status, Start/End dates, Notes, Icon, SupervisorSUPERVISOR + ADMIN
RACI (R/A/C/I)Μόνο ADMIN
ON_GOINGΜόνο ADMIN

Στη φόρμα υπάρχουν επίσης: Συνημμένα, Σχόλια, ενότητα Υπογραφή (Signed On / Signed By, read-only) και το «Record information» (ίχνη δημιουργίας/τροποποίησης).

Project σε άλλους ρόλους πλην των παραπάνω ανοίγει μόνο για ανάγνωση. Διαγραφή project: μόνο ADMIN (τα tasks δεν διαγράφονται, αποσυνδέονται).
Ενότητα 5

Κύκλος Ζωής Project & Υπογραφή

Κάθε αλλαγή status περνά από κανόνες — ίδιοι είτε γίνεται από τη φόρμα είτε με drag-n-drop στο Kanban.

Σχηματική ροή

In Progress

Οι πλάγιες καταστάσεις (Deferred, Waiting) επιτρέπονται μόνο από/προς In Progress (και μεταξύ τους κατά τους κανόνες).

Η υπογραφή (Signoff) του project

  1. Το νέο project γεννιέται σε Waiting Manager Approval και ο Supervisor ειδοποιείται με email.
  2. Ο SUPERVISOR του έργου (ή ο ADMIN) πατά ✍ Υπογραφή στη φόρμα. Τότε αυτόματα: Signed On = τώρα, Signed By = ο υπογράφων, status → Not Started, και προτείνονται defaults: Accountable = το Resource του Supervisor, Consulted = το Resource του Owner (αν είναι κενά).
  3. Αν δεν υπάρχει κανένα συνημμένο, εμφανίζεται προειδοποίηση («το έργο δεν περιγράφεται επαρκώς») — με ΟΚ η υπογραφή προχωρά.
Χωρίς υπογραφή, το status δεν μπορεί να τεθεί «Not Started». Υπογεγραμμένο project δεν επιστρέφει ποτέ σε «Waiting Manager Approval».

Κανόνες μετάβασης (σύνοψη)

Νέο statusΕπιτρέπεται μόνο απόΑυτοματισμοί / έλεγχοι
Not Startedμέσω ΥπογραφήςSigned On/By, RACI defaults
In ProgressNot Started, Deferred, WaitingΑπαιτεί όλα τα RACI συμπληρωμένα · Start date = σήμερα αν κενό
CompletedIn Progress, Deferred, WaitingΑν υπάρχουν ανοιχτά tasks → ερώτηση επιβεβαίωσης · End date = σήμερα αν κενό
DeferredIn Progress, Waiting—
Waiting on someone elseIn Progress, Deferred—

Σφάλματα και προειδοποιήσεις εμφανίζονται σε παράθυρο με κουμπί OK: σε σφάλμα η αλλαγή δεν γίνεται και επιστρέφετε για διόρθωση· σε προειδοποίηση, με το ΟΚ η ροή συνεχίζει.

Οι κανόνες εγγυώνται ότι κανένα έργο δεν «τρέχει» χωρίς έγκριση, ομάδα RACI και σωστές ημερομηνίες.
Ενότητα 6

Tasks

Οι επιμέρους εργασίες: ποιος τις βλέπει, ποιος τις επεξεργάζεται, και οι κανόνες status.

Λίστα Tasks (grid)

Στήλες: # (Task ID), Τίτλος, Project, Tag, Status, Assigned to, Priority, ημερομηνίες, Sign Off, flags. Πλήρη φίλτρα, chips (Open, Overdue, All…), Export Excel.

Ποια tasks βλέπει ο καθένας
ADMINΌλα, με πλήρη επεξεργασία. Μόνο αυτός δημιουργεί «κανονικά» tasks (εκτός ON_GOING — βλ. ενότητα 7).
OWNERΤα tasks των δικών του projects — μόνο προβολή.
SUPERVISORΤα tasks των εποπτευόμενων projects — μόνο προβολή (πλην ροής SignOff).
IMPLEMENTORΤα ανατεθειμένα σε αυτόν· επεξεργάζεται πεδία εκτέλεσης: Status, ημερομηνίες, % Complete, ώρες, Implementor notes, Project, Tag.

Κανόνες Status (Task Status Rules)

Νέο statusΈλεγχοι & αυτοματισμοί
Not StartedΠροεπιλογή για νέο task.
In Progress · Deferred · Waiting❌ Μπλοκάρεται αν Unassigned · Actual Start = σήμερα αν κενό · ⚠ Warning αν λείπουν Estimated hours ή Due date (η αποθήκευση προχωρά).
Completed❌ Μπλοκάρεται αν Unassigned · Actual Start & End = σήμερα αν κενά · ⚠ Warning αν λείπουν Estimated ή Billed hours.

Οι κανόνες ισχύουν παντού: φόρμα και Kanban drag-n-drop.

Διαγραφή task

ΚατάστασηΠοιος διαγράφει
Πριν το SignOffΟ δημιουργός (Created By) ή ο ADMIN
Μετά το SignOff, status = Not StartedΟ τελευταίος που το τροποποίησε (Modified By) ή ο ADMIN
Το status έχει προχωρήσειΜόνο ο ADMIN

Χρήσιμα κουμπιά στο task

  • 📅 Add Outlook — δημιουργεί προσυμπληρωμένο Outlook event στη Due Date 17:00 με τίτλο «Deadline: <task> | <project>». Το αποθηκεύετε εσείς.
  • ✉ Έγκριση Προϊσταμένου — ξανα-ανοίγει το email έγκρισης προς τον Assigned to manager (βλ. ενότητα 7).
  • Comments tab & συνημμένα — κοινά με το SharePoint.
Το task «κουβαλάει» όλη την ιστορία του: ποιος το ζήτησε, ποιος το ενέκρινε, σχόλια, αρχεία, εκδόσεις.
Ενότητα 7

ON_GOING Projects & η Μίνι-Εγκριτική Ροή

Projects-«κουβάδες» για τη συνεχή, καθημερινή δουλειά (π.χ. «Συντήρηση/bug fixing ERP») — δεν κλείνουν ποτέ.

Τι είναι

  • Πεδίο ON_GOING = TRUE στο project — το ορίζει και το αλλάζει μόνο ο ADMIN.
  • Χωρίς πρακτικό deadline: μπαίνει αυτόματα 01/01/2999.
  • Ορατά σε όλους τους χρήστες στη λίστα Projects (με badge ON-GOING), αλλά editable μόνο από Admin.
  • Εξαιρούνται από τις 3 οθόνες του Projects Overview — δεν «μολύνουν» τα στατιστικά των κανονικών έργων.
Τα πολλά μικρά καθημερινά αιτήματα (CRs, bug fixes) αποκτούν στοιχειώδη έγκριση και καταγραφή, χωρίς τη γραφειοκρατία ενός κανονικού project.

Η εξαίρεση: όλοι ανοίγουν task σε ON_GOING project

Κατά κανόνα, Owners & Supervisors δεν δημιουργούν tasks. Εξαίρεση: σε ON_GOING projects, task μπορεί να ανοίξει οποιοσδήποτε — και ακολουθεί η μίνι-εγκριτική ροή:

1. Ο χρήστης ανοίγει task
(Title, Due, Notes, Project,
Assigned to = manager)
→✉ email «Έγκριση Task #»
2. Ο Supervisor το ελέγχει,
επισυνάπτει οδηγίες,
πατά SignOff
→✉ email «Εγκεκριμένο Task #» στον COO
3. Ο COO αναθέτει
στον implementor —
κανονική διαχείριση

Βήμα 1 — Απλός χρήστης OWNER / IMPLEMENTOR

  • Ανοιχτά πεδία ΜΟΝΟ: Title, Due Date, Requestor Notes, Project (μόνο ON_GOING projects στη λίστα) και Assigned to — όλα υποχρεωτικά.
  • Το Assigned to δείχνει μόνο managers (Resources με Is_Manager): επιλέγετε ποιος θα εγκρίνει.
  • Με την αποθήκευση ανοίγει email προς τον manager αυτόν, θέμα «Έγκριση Task #N <τίτλος>». Για επανάληψη υπάρχει πάντα το κουμπί Έγκριση Προϊσταμένου στην προβολή του task.

Βήμα 2 — SUPERVISOR (ή ADMIN)

  • Επεξεργάζεται Title, Due Date, Requestor Notes, Project και επισυνάπτει έγγραφα-οδηγίες.
  • Δίπλα στο Assigned to υπάρχει κουμπί SignOff — ενεργό μόνο όσο το SignOff είναι κενό. Με το πάτημα: SignOff = τώρα (καταγράφεται στο Record information & στο grid).
  • Στην αποθήκευση: ⚠ warning αν δεν υπάρχουν συνημμένες οδηγίες (με ΟΚ προχωρά) και ανοίγει email προς τον COO: «Εγκεκριμένο Task #N <τίτλος>».
  • Αν το task το ανοίξει εξαρχής Supervisor, το Βήμα 1 (email έγκρισης) παραλείπεται.
Μετά το SignOff, το task αντιμετωπίζεται όπως όλα τα υπόλοιπα (Kanban, Gantt, status rules, αναθέσεις).
Ενότητα 8

Οθόνη Waiting for Signoff

Η προσωπική ουρά υπογραφών — διαθέσιμη σε ADMIN και SUPERVISOR (τελευταία επιλογή του μενού).

Τμήμα 1 — Projects για υπογραφή

  • Λίστα των μη υπογεγραμμένων projects (Signed On κενό).
  • Toggle: Waiting Manager Approval (η τρέχουσα ουρά — προεπιλογή) / Όλα τα ανυπόγραφα (και τα παλιά που προχώρησαν χωρίς υπογραφή).
  • Κλικ σε γραμμή → ανοίγει η φόρμα του project σε edit, όπου βρίσκεται το κουμπί ✍ Υπογραφή.
Διαφοροποίηση
ADMINΒλέπει όλα τα ανυπόγραφα.
SUPERVISORΜόνο όσα περιμένουν τη δική του υπογραφή (είναι ο Supervisor τους).

Τμήμα 2 — Tasks για υπογραφή

  • Tasks με Assigned To = εσάς και SignOff κενό — αυτά που σας έστειλαν για έγκριση.
  • Toggle: ON_GOING projects (προεπιλογή) / Όλα.
  • Κλικ σε γραμμή → προβολή του task· από εκεί Edit για τη ροή SignOff.
Μηδέν ξεχασμένες εγκρίσεις: ό,τι περιμένει την υπογραφή σας, σε μία οθόνη.
Ενότητα 9

Overviews (Kanban · Dashboard · RACI · Gantt)

Οι εποπτικές οθόνες του ADMIN — συνολική εικόνα έργων και εργασιών. (Οι έννοιες εξηγούνται στην ενότητα 2.)

Projects Overview — 3 views με ΚΟΙΝΑ φίλτρα

  • Kanban: κάρτες ανά status, με drag-n-drop αλλαγή status (εφαρμόζονται οι κανόνες της ενότητας 5 — άκυρη μετάβαση = μήνυμα και η κάρτα δεν μετακινείται).
  • Dashboard: KPIs (Total/Active/Completed/Overdue) και κατανομές ανά status & owner.
  • RACI: πίνακας projects × resources με τα γράμματα R/A/C/I ανά κελί + Export Excel.

Τα φίλτρα (Owner, Approver, Status, Resource, R, A, C, I, Deadline) είναι κοινά και διατηρούνται όταν εναλλάσσεστε μεταξύ των 3 views. Το φίλτρο Resource είναι «έξυπνο»: φέρνει τα projects όπου το επιλεγμένο resource συμμετέχει οπουδήποτε — ως R, A, C, I ή ως πρόσωπο Supervisor/Owner. Κουμπί «Αναίρεση Φίλτρων» καθαρίζει τα πάντα με ένα κλικ.

Τα ON_GOING projects εξαιρούνται από τα 3 views.

Tasks Overview

  • Kanban: στήλες ανά status με drag-n-drop (με τους Task Status Rules) και φίλτρα Assignee/Tag/Project/Deadline.
  • Gantt: τα tasks στον χρόνο.
  • Dashboard: συγκεντρωτικοί δείκτες εργασιών.
Ένα σύρσιμο κάρτας = ενημερωμένο status με όλους τους ελέγχους — χωρίς να ανοίξετε καμία φόρμα.
Ενότητα 10

Κοινές Λειτουργίες

Ό,τι δουλεύει με τον ίδιο τρόπο σε όλες τις οθόνες, για όλους τους ρόλους.

Grids (λίστες)

  • Ταξινόμηση: κλικ στην επικεφαλίδα στήλης.
  • Φίλτρα ανά στήλη: βελάκι δίπλα στην επικεφαλίδα — checkbox τιμών, κείμενο ή ημερομηνίες (προκαθορισμένα διαστήματα, από–έως).
  • Chips γρήγορου φιλτραρίσματος πάνω από το grid (π.χ. ανά status) και πεδίο Αναζήτηση.
  • Σελιδοποίηση (10 γραμμές) και ⬇ Export Excel με τα τρέχοντα φιλτραρισμένα δεδομένα.
  • Κόκκινη σήμανση = εκπρόθεσμο. Το grid προσαρμόζει το ύψος του στην οθόνη με σταθερές επικεφαλίδες.

Συνημμένα & Σχόλια

  • Σε κάθε project/task: μεταφόρτωση αρχείων και σχόλια με όνομα & ώρα. Διαγράφετε μόνο τα δικά σας σχόλια.
  • Όλα συγχρονίζονται με το SharePoint — ίδια δεδομένα κι από τις δύο πλευρές.

Emails

Η εφαρμογή δεν στέλνει ποτέ email «μόνη της»: ανοίγει προσυμπληρωμένο μήνυμα στο δικό σας Outlook (παραλήπτης/θέμα) και το στέλνετε εσείς — π.χ. signoff project, έγκριση task, Mail-2-Team, Add Outlook (ημερολόγιο).

Μηνύματα σφαλμάτων & προειδοποιήσεων

  • ⛔ Σφάλμα: η ενέργεια δεν εκτελείται· με το ΟΚ επιστρέφετε στο σημείο διόρθωσης.

Ενότητα 11

Γλωσσάρι

Project
Έργο με Owner, Supervisor, deadline και ομάδα RACI. Ομαδοποιεί tasks.
Task
Επιμέρους εργασία/αίτημα, συνήθως συνδεδεμένο με project και ανατεθειμένο σε implementor.
Owner (Requestor)
Ο αιτών/κάτοχος ενός project — το άνοιξε και το παρακολουθεί.
Supervisor (Manager)
Ο προϊστάμενος που εγκρίνει: υπογράφει projects και κάνει SignOff σε tasks ON_GOING.
Implementor (Resource)
Ο υλοποιητής που εκτελεί tasks και ενημερώνει την πρόοδό τους.
Admin / COO
Ο διαχειριστής της εφαρμογής με πλήρη δικαιώματα και τις εποπτικές οθόνες.
RACI
Responsible, Accountable, Consulted, Informed — οι 4 ρόλοι ομάδας ενός έργου (βλ. ενότητα 2).
Kanban
Πίνακας με στήλες ανά status· οι κάρτες σέρνονται από στήλη σε στήλη.
Gantt
Χρονοδιάγραμμα με οριζόντιες μπάρες διάρκειας ανά task.
Dashboard
Οθόνη συγκεντρωτικών δεικτών (KPIs) και κατανομών.
Status
Η κατάσταση project/task: Waiting Manager Approval, Not Started, In Progress, Waiting on someone else, Deferred, Completed κ.ά.
Υπογραφή (Project Signoff)
Η έγκριση project από τον Supervisor: Signed On/By + αυτόματο πέρασμα σε Not Started.
SignOff (Task)
Ημερομηνία/ώρα έγκρισης ενός task από manager, στη μίνι-εγκριτική ροή των ON_GOING.
ON_GOING project
Project-«κουβάς» συνεχούς εργασίας, χωρίς πρακτικό deadline (01/01/2999), που δεν κλείνει ποτέ.
Due Date / Deadline
Η καταληκτική ημερομηνία task / project αντίστοιχα.
Estimated / Billed hours
Εκτιμώμενες / πραγματικές (χρεωθείσες) ανθρωποώρες ενός task.
% Complete
Ποσοστό ολοκλήρωσης task (0–100%).
Tag
Ετικέτα κατηγοριοποίησης task (με χρώμα).
Activity Feed
Ο «τοίχος» σχολίων της Welcome Screen — όλα τα σχόλια projects & tasks χρονολογικά.
Overdue
Μη ολοκληρωμένο στοιχείο του οποίου η καταληκτική ημερομηνία έχει περάσει.
Role preview
Δυνατότητα του Admin να βλέπει την εφαρμογή όπως τη βλέπει κάθε άλλος ρόλος.
